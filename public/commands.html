Port Scanning & Service Checks

for port in 8089 9997 8088 514; do nc -zv host $port 2>&1; done

SSL / TLS Inspection

Certificate & SSL Verification

bash 
# Check SSL certificate details
openssl s_client -connect splunk-idx.company.com:8089 </dev/null 2>/dev/null | openssl x509 -noout -dates -subject

# Verify certificate chain
openssl s_client -connect splunk-idx.company.com:8089 -showcerts </dev/null 2>&1 | grep -E "subject|issuer|depth"

# Check cipher suites supported
nmap --script ssl-enum-ciphers -p 8089 splunk-idx.company.com

# Test TLS version
openssl s_client -connect splunk-idx.company.com:8089 -tls1_2 </dev/null 2>&1 | head -5
Packet Capture

tcpdump & Network Analysis

bash 
# Capture traffic on specific port
sudo tcpdump -i any port 9997 -c 100 -w /tmp/splunk_fwd.pcap

# Filter by host and port
sudo tcpdump -i eth0 host 10.0.1.50 and port 514 -nn

# Show packet content (ASCII)
sudo tcpdump -i any port 8088 -A -c 20

# Capture HEC traffic
sudo tcpdump -i any port 8088 -w /tmp/hec_capture.pcap -G 60 -W 1

# Read pcap file
tcpdump -r /tmp/splunk_fwd.pcap -nn | head -20

Regex Cheat Sheet

Character Classes

.Any character (except newline)
\dDigit [0-9]
\DNon-digit
\wWord char [a-zA-Z0-9_]
\WNon-word char
\sWhitespace
\SNon-whitespace
[abc]Any of a, b, or c
[^abc]Not a, b, or c
[a-z]Range: a to z

Anchors

^Start of string/line
$End of string/line
\bWord boundary
\BNot word boundary

Groups

(abc)Capture group
(?P<name>abc)Named capture (Python/Splunk)
(?:abc)Non-capturing group
(a|b)Alternation (or)
\1Backreference to group 1

Quantifiers

*0 or more (greedy)
+1 or more (greedy)
?0 or 1 (optional)
{n}Exactly n times
{n,m}Between n and m times
*?0 or more (lazy)
+?1 or more (lazy)

Lookahead / Lookbehind

(?=abc)Positive lookahead
(?!abc)Negative lookahead
(?<=abc)Positive lookbehind
(?<!abc)Negative lookbehind

Splunk Rex Tips

| rex field=_raw "(?<name>pattern)"
Named groups become Splunk fields
| rex mode=sed "s/old/new/g"
Sed mode for find/replace
| regex _raw="pattern"
Filter events by regex match

RFC 3164 (BSD Syslog)

BSD Syslog Format

Format: <priority>timestamp hostname app[pid]: message

syslog 
<134>Mar 02 14:30:00 webserver01 nginx[1234]: GET /api/health 200 0.003
<131>Mar 02 14:30:01 fw01 iptables: DROP IN=eth0 SRC=192.168.1.100 DST=10.0.0.1 PROTO=TCP DPT=22
RFC 5424 (Structured Syslog)

Structured Syslog Format

Format: <priority>version timestamp hostname app-name procid msgid [structured-data] message

syslog 
<134>1 2024-03-02T14:30:00.000Z webserver01 nginx 1234 - - GET /api/health 200
<165>1 2024-03-02T14:30:01.000Z fw01 iptables 5678 - [meta src="192.168.1.100"] DROP IN=eth0

Curated Conf Reference (12 common files)
▶ inputs.conf Data inputs: file, network, scripted, HEC, modular
Defines all data input methods. Each stanza type handles a different input mechanism. Located in $SPLUNK_HOME/etc/system/local/ or app-specific directories.
Key Options
[monitor://<path>]Monitor files/directories for new data
[tcp://<port>]Receive data on TCP port (raw)
[udp://<port>]Receive data on UDP port
[splunktcp://<port>]Receive from Splunk forwarders (S2S protocol)
[http://<name>]HTTP Event Collector token
[script://<path>]Scripted input (runs at interval)
index = <name>Target index for this input
sourcetype = <name>Set sourcetype for events
disabled = true|falseEnable/disable this input
whitelist/blacklistRegex patterns for file include/exclude
Best Practices
• Always set index= and sourcetype= explicitly
• Use whitelist/blacklist for directory monitors
• Set disabled=false explicitly for clarity
• Use HEC for cloud/container workloads
• Monitor /var/log/ not /var/log/* (Splunk handles recursion)
[monitor:///var/log/syslog] sourcetype = syslog index = os disabled = false [http://my_hec_token] token = <guid> index = main sourcetype = json disabled = false
▶ props.conf Parsing, timestamping, line breaking, field extraction
Controls how Splunk parses incoming data: timestamp extraction, line merging, character encoding, field extraction, and transforms references. The most commonly edited conf file.
Key Options
TIME_FORMAT = <strftime>strftime format for timestamp parsing
TIME_PREFIX = <regex>Regex before timestamp in event
MAX_TIMESTAMP_LOOKAHEADCharacters to scan for timestamp (default 150)
LINE_BREAKER = <regex>Regex for event boundary (capture group = break)
SHOULD_LINEMERGEtrue|false — merge lines into events (default true)
BREAK_ONLY_BEFORERegex — break before matching line
EXTRACT-<name>Inline regex field extraction (search-time)
REPORT-<name>Reference to transforms.conf stanza
TRANSFORMS-<name>Index-time transforms reference
LOOKUP-<name>Automatic lookup reference
KV_MODE = auto|none|json|xmlKey-value extraction mode
TZ = <timezone>Force timezone for events (e.g., US/Eastern)
Best Practices
• Set SHOULD_LINEMERGE=false for known single-line formats
• Always define TIME_FORMAT explicitly (don't rely on auto-detect)
• Use LINE_BREAKER for multi-line events (Java stack traces, XML)
• Test with the Conf Simulator before deploying
• Use REPORT- for reusable extractions, EXTRACT- for sourcetype-specific
▶ transforms.conf Extractions, lookups, routing, field aliases
Defines reusable field extractions, lookup definitions, event routing rules, and field aliases. Referenced by props.conf via REPORT-, TRANSFORMS-, and LOOKUP- stanzas.
REGEX = <pcre>Perl-compatible regex for extraction
FORMAT = field::$NField assignment from capture groups
DEST_KEY = _MetaData:IndexEvent routing destination (index-time)
filename = <csv>Lookup definition — CSV file in lookups/
external_cmd = <script>External lookup script
Best Practices
• Use named capture groups: (?P<field_name>...)
• Test REGEX in the Regex Tester tool before deploying
• Use REPORT- (search-time) not TRANSFORMS- (index-time) unless routing
• Keep lookup CSVs under 50MB for performance
▶ savedsearches.conf Saved searches, alerts, reports, dashboards
Defines all saved searches, scheduled reports, alerts, and their actions. Each stanza is a saved search name. Controls scheduling, triggering, and alert actions.

• Stagger cron schedules to avoid search concurrency limits
• Set dispatch.latest_time relative to cron for consistent windows

• Throttle alerts with alert.suppress settings

Search Best Practices
Do (fast searches)
index=main sourcetype=syslog
index::main (bloom filter, fastest)
index IN (main, web, security)
earliest=-1h latest=now
| stats count by host (streaming)
| tstats count where index=main (tsidx)
| fields - _raw (drop early)
Avoid (expensive patterns)
index=* (scans ALL indexes)
NOT index=_internal (still scans all)
| join (use stats/lookup instead)
| transaction (memory-intensive)
| append [search ...] (2 searches)
| where match(_raw, "...") (filter in search)
| table * (no field projection)
